LOGICA GENERAL
--------------

Al abrir la forma:
  1. Leer PA.PARAM_GENERALES donde param = 'ANTECEDENTES_CRED' (formato '|COD1|COD2|...')
  2. Si :GLOBAL.CodEmpresa ESTA en el parametro:
       - Modo multi-empresa: mostrar listbox con todas las empresas del parametro
       - Al cambiar seleccion: actualizar :variables.CodEmpresa y recargar bloques
  3. Si :GLOBAL.CodEmpresa NO esta en el parametro:
       - Comportamiento actual sin ningun cambio

3. NUEVO PROCEDIMIENTO LOCAL: VERIFICA_ANTECRED
------------------------------------------------

NOTA: La deteccion de modalidad usa la funcion de base de datos
      pr.empresa_en_antec_cred(vp_cod_empresa) ya disponible
      (ver funcion_modalidad.sql). Retorna TRUE si la empresa
      figura en el parametro ANTECEDENTES_CRED, FALSE si no.
      No es necesario releer el parametro ni hacer INSTR manualmente.

PROCEDURE verifica_antecred IS
  v_param  VARCHAR2(500);
  v_en_ant BOOLEAN;
  v_idx    NUMBER := 1;
BEGIN
  -- Delegar la deteccion a la funcion de BD ya existente
  v_en_ant := pr.empresa_en_antec_cred(:variables.CodEmpresa);

  IF v_en_ant THEN

    -- Empresa actual esta en el parametro -> modo multi-empresa
    :variables.multi_empresa := 'S';

    -- Leer el parametro solo para poblar el listbox (uso local, no se persiste)
    v_param := pa.parametro_general('PR', 'ANTECEDENTES_CRED');

    -- Poblar el listbox con las empresas del parametro
    CLEAR_LIST('control.lst_empresa');
    FOR reg IN (
      SELECT e.cod_empresa, e.nombre
        FROM pa.empresa e
       WHERE INSTR(v_param, '|' || e.cod_empresa || '|') > 0
       ORDER BY e.cod_empresa
    ) LOOP
      ADD_LIST_ELEMENT('control.lst_empresa', v_idx,
                       reg.nombre, reg.cod_empresa);
      v_idx := v_idx + 1;
    END LOOP;

    -- Seleccionar la empresa actual por defecto
    SET_LIST_ELEMENT_VALUE('control.lst_empresa', :variables.CodEmpresa);
    SET_ITEM_PROPERTY('control.lst_empresa', VISIBLE, PROPERTY_TRUE);

  ELSE

    -- Empresa NO esta en el parametro -> modo normal sin cambios
    :variables.multi_empresa := 'N';
    SET_ITEM_PROPERTY('control.lst_empresa', VISIBLE, PROPERTY_FALSE);

  END IF;
END;

Agregar al final del procedimiento, luego de la llamada a IniVarGlobalesLocales:

FUNCION DE MODALIDAD DISPONIBLE EN BD
--------------------------------------

  pr.empresa_en_antec_cred(vp_cod_empresa IN VARCHAR2) RETURN BOOLEAN

  Retorna TRUE  si la empresa figura en ANTECEDENTES_CRED.
  Retorna FALSE si es NULL o no figura.
  Encapsula la lectura del parametro y el INSTR, evitando duplicar
  esa logica en el codigo Forms.
  Usar esta funcion como unica fuente de verdad para la deteccion
  de modalidad, tanto en la forma como en cualquier otro objeto de BD.


POR QUE LOS PROCEDIMIENTOS DE CONSULTA NO NECESITAN CAMBIOS
------------------------------------------------------------

Todos los cursores ya filtran con :variables.CodEmpresa.
Basta actualizar esa variable desde el trigger WHEN-LIST-CHANGED
para que las queries devuelvan datos de la empresa seleccionada.

  OBTIENE_TRAMITES   -> PR_TRAMITE.cod_empresa = :variables.CodEmpresa
  OBTIENE_TRAMITES2  -> igual
  OBT_RECIPROCIDAD   -> a.cod_empresa = :variables.codempresa (x5)
  OBT_GAR_CLI        -> idem
  LOVs/Record Groups -> idem
